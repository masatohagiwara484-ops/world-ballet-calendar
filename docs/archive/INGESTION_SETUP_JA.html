0このシステムは何をするのか

これまで手で入れていた公演データを、AIが世界中の劇場サイトを自動巡回して見つけ、あなたのスマホ（Telegram）に「これ載せていい？」と聞いてくる仕組みです。あなたは ✅ 承認 / 🚫 却下 をタップするだけ。承認したものだけがサイトに載ります。

① GitHub Actions が2日に1回 自動起動
VPS（自分のサーバー）は不要
↓
② 各劇場サイトを巡回
feed優先、無ければAIが読み取り
↓
③ 新規・変更・消滅を検出 → 「承認待ち」で保存
review_status = pending
↓
④ あなたの Telegram に劇場ごとのまとめが届く
↓
⑤ あなたが ✅ / 🚫 をタップ
↓
⑥ 承認分だけ公開 → 数分でサイト反映
💡 GitHub Actions＝GitHubが無料で提供する「自動実行ロボット」。決めた時刻にプログラムを動かせる（VPS不要）。
feed（フィード）＝劇場が公式に出す「データの蛇口」（iCal/RSS等）。
Telegram＝無料のメッセージアプリ。ここに通知が届く。

1Supabase の準備（SQL 3つ）

新プロジェクトは作成済みなので、ここでは SQLを順に流す作業をします。

  1. Supabase → 左メニュー 「SQL Editor」 → 「+ New query」
  2. 次の3ファイルを、この順番で1つずつ貼り付けて Run：
① supabase/migrations/002_rebuild_schema.sql  （基本の箱）
② supabase/migrations/003_entity_graph.sql    （横断検索の箱）
③ supabase/migrations/004_ingestion.sql       ★今回の新システム用
⚠️ 必ず 002 → 003 → 004 の順番で実行。逆だとエラーになります。

終わったら 「Project Settings（歯車）→ API」 で次の3つをメモ：Project URL / anon public / service_role🔒

2環境変数 7個 — 鍵の置き場所

2-1. 7つの鍵と置き場所

#環境変数名置く場所意味・取得先
1NEXT_PUBLIC_SUPABASE_URLVercel ＋ .env.localProject URL 公開OK
2NEXT_PUBLIC_SUPABASE_ANON_KEYVercel ＋ .env.localanon public 公開OK
3SUPABASE_SERVICE_ROLE_KEYGitHub Secrets ＋ Vercel ＋ .env.localservice_role 極秘
4ANTHROPIC_API_KEYGitHub Secrets ＋ .env.localAIの鍵。月予算を設定 極秘
5TELEGRAM_BOT_TOKENGitHub Secrets ＋ Vercel@BotFatherで作成 TG
6TELEGRAM_CHAT_IDGitHub Secrets ＋ Vercel@userinfobotで取得 TG
7TELEGRAM_WEBHOOK_SECRETVercel自分で決める合言葉 TG
💡 .env.local＝自分のPC内だけの秘密ファイル（ローカル実行用）。
GitHub Secrets＝自動巡回ロボット（Actions）が使う鍵の金庫。
Vercel＝本番サイト／Telegram通知の受け取り用。
⚠️ 同じ鍵を複数の場所に入れるものがあります。面倒でも表の通り全部に入れてください。

2-2. .env.local の作り方（PC側・4つ）

ファイル名 .env.local を作り、本物の値に書き換えて保存：

NEXT_PUBLIC_SUPABASE_URL=https://あなたのID.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=anonキーを貼る
SUPABASE_SERVICE_ROLE_KEY=service_roleキーを貼る
ANTHROPIC_API_KEY=sk-antで始まるキーを貼る

確認：cat .env.local。Telegramの3つは .env.local には不要（Secrets と Vercel に入れる）。

2-3. GitHub Secrets（ロボット用・4つ）

  1. GitHub → Settings（歯車）
  2. 左メニュー Secrets and variables → Actions
  3. 緑ボタン New repository secret
  4. Name に変数名・Secret に値 → Add secret
  5. 登録：SUPABASE_SERVICE_ROLE_KEY / ANTHROPIC_API_KEY / TELEGRAM_BOT_TOKEN / TELEGRAM_CHAT_ID

2-4. Vercel（本番サイト用・6つ）

  1. Vercel → このプロジェクト → Settings → Environment Variables
  2. Key と Value を入れて Save
  3. 登録：URL / ANON / SERVICE_ROLE / TELEGRAM_BOT_TOKEN / TELEGRAM_CHAT_ID / TELEGRAM_WEBHOOK_SECRET
  4. 入れ終わったら Deployments → 最新を Redeploy（再デプロイ）で反映

2-5. ANTHROPIC_API_KEY の取り方

  1. console.anthropic.com にログイン
  2. API Keys → Create Key（sk-ant-...）
  3. Billing で予算上限と使用量アラートを必ず設定（暴走防止。月$5など低めでOK）

3seed ＋ verify（最初のデータ投入）

⚠️ 第1章（SQL）と第2章（.env.local）が終わってから。ターミナル操作です。 
cd /home/user/world-ballet-calendar
npm install            # 初回だけ
npm run seed           # 25団体・約155公演を投入
npm run validate:data  # 検証（緑ならOK）
💡 初期データは最初から「公開」状態なので、サイトはすぐ埋まり空になりません。自動巡回で新たに見つかった分だけが「承認待ち」になります。

4Telegram bot の設定（コマンドあり）

承認ボタンを押せるようにする設定。順にやれば約10分です。

4-1. bot を作って TOKEN を取る

  1. Telegram で @BotFather を開く
  2. /newbot を送信 → 名前とユーザー名を決める
  3. 表示される長い文字列が TELEGRAM_BOT_TOKEN → GitHub Secrets と Vercel に登録

4-2. 自分の chat_id を取る

  1. Telegram で @userinfobot を開いて何か送信
  2. 返ってくる数字（例 123456789）が TELEGRAM_CHAT_ID → Secrets と Vercel に登録

4-3. WEBHOOK_SECRET（合言葉）を決める

適当なランダム文字列（例 my-secret-9f3a7c2b）を自分で決める → これが TELEGRAM_WEBHOOK_SECRET → Vercel に登録。

💡 Webhook（ウェブフック）＝「ボタンが押されたよ」という通知を Telegram からあなたのサイトへ自動で送る仕組み。合言葉で本物か確認します。

4-4. Webhook を登録する（コマンド1回だけ）

Vercel にデプロイした後で、<...> を自分の実際の値に置き換えて実行：

curl "https://api.telegram.org/bot<BOT_TOKEN>/setWebhook" \
  -d "url=https://worldballetoperacalender.vercel.app/api/telegram/webhook" \
  -d "secret_token=<WEBHOOK_SECRET>"
💡 curl（カール）＝ターミナルから指定URLに命令を送るコマンド。これは「ボタンが押されたらこのサイトに知らせて」という1回きりの登録。成功すると {"ok":true,...} と返ります。VPSは不要です。

5取材元 SOURCES の登録

5-1. まず 10〜15館を選んで調べる

docs/SOURCES_FILL_IN_JA.md で4項目を埋める：Listing URL / Feed / Affiliate / robots/ToS。

⚠️ 各劇場の robots.txt と利用規約を必ず確認。禁止なら forbidden → ロボットが自動でその館を飛ばします。公式feed・アフィリエイトAPIがあれば最優先。

5-2. 巡回ロボットに登録する（コード1行）

scripts/ingest/run-ingest.ts の SOURCES に追記：

'royal-ballet': { companySlug: 'royal-ballet', url: '<公演一覧URL>', kind: 'ical' },
  • 'ical' / 'rss' / 'jsonld' → feedがある場合。AI不使用・無料・正確。
  • 'html' → feedが無い場合。AI（Haiku）がページを読む。
💡 この1行追記が不安なら、調べた表を Claude Code に貼って「run-ingest.ts に登録して」と頼めばこちらで行います。

5-3. 自動承認について

  • ある館で 3回連続で問題なく承認すると、その館は自動承認に昇格（手間が減る）。
  • 1回でも却下するとリセット。
  • 日付変更・公演中止だけは必ず毎回あなたの手動承認（信用を守るため）。

6実行する（動かし方）

やりたいこと操作
手動で今すぐ巡回GitHub → Actions → Ingest → Run workflow
同上（ターミナル）npm run ingest -- --all --live

実行すると 劇場ごとに1通のまとめ通知が Telegram に届きます。✅ Approve all か 🚫 Reject all をタップ。承認分は数分でサイト反映（トップ・検索・該当劇場・作品・人物ページが自動更新）。

7VPS は使いません

💡 VPS（ブイピーエス）＝自分専用のレンタルサーバー。
  • 自動巡回 → GitHub Actions（2日に1回）が担当。
  • 承認ボタンの受け取り → Vercel上のサイト（/api/telegram/webhook）が担当。
  • どちらも無料〜低額枠で動くため、VPSの契約・管理はオーナーには発生しません。
